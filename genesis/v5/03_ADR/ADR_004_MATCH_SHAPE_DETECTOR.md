# ADR 004: Детектор Фигур и Создание Специальных Сфер (Match Shape Detector)

## Статус
Принято (Active)

## Контекст
Match-3 игры строят свой геймплей вокруг создания специальных элементов из сложных фигур. Требуется строгий математический распознаватель геометрических шаблонов (линии, квадраты, L/T-формы), который работает за линейное время $O(N)$ без лишних аллокаций памяти и корректно ранжирует фигуры по приоритету (например, если совпали одновременно линия-4 и квадрат 2x2).

## Решение
1. **Математический Детектор MatchShapeDetector**:
   Создать класс `MatchShapeDetector`, который сканирует логическую сетку поля. Сканирование происходит в два этапа:
   - *Этап 1 (Поиск базовых линий)*: Обнаружение всех горизонтальных и вертикальных линий длиной 3 и более.
   - *Этап 2 (Слияние пересечений и классификация)*: Группировка пересекающихся линий и паттернов для определения сложных фигур:
     - `Line-3`: 3 сферы в линию.
     - `Line-4`: 4 сферы в линию.
     - `Square-2x2`: 4 сферы в виде квадрата 2x2.
     - `L-shape` / `T-shape`: Две перпендикулярные линии, сходящиеся углом или пересекающиеся.
     - `Cross '+'`: Пересечение двух линий ровно посередине.
     - `Line-5`: 5 сфер в линию.
     - `Complex-6`: Группа из 6 сфер.
     - `Complex-7+`: Группа из 7 и более сфер.

2. **Ранжирование и Веса фигур**:
   Каждая фигура имеет уникальный вес для расчета очков и приоритета создания спец-элементов (см. Shape Detection Matrix в PRD). При перекрытии фигур приоритет отдается той, у которой выше вес.

3. **Размещение Спец-сфер**:
   - Спец-сфера спавнится в ячейке, которая была инициатором хода (куда игрок сделал свайп).
   - Если матч произошел в результате каскада (без прямого участия игрока), спец-сфера спавнится в геометрическом центре совпавшей фигуры (или на пересечении линий для T/L фигур).

4. **Матрица Спец-комбинаций (Special Combination Matrix)**:
   При свайпе двух смежных спец-сфер срабатывают мощные комбинированные эффекты, которые переопределяют обычное поведение взрыва. Эти правила являются data-driven и настраиваются через JSON-конфиг.

## Последствия
- Точное соответствие механикам ведущих Match-3 игр.
- Быстрый расчет без утечек памяти и зависаний в циклах.
- Data-driven конфигурация спец-комбинаций упрощает балансировку.
