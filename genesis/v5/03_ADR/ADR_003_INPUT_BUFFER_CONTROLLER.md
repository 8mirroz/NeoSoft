# ADR 003: Ввод во время каскадов и Буферизация (Input Buffer Controller)

## Статус
Принято (Active)

## Контекст
Игроки в динамичные Match-3 игры часто делают быстрые ходы "на опережение", свайпая сферы, которые всё ещё находятся в процессе падения. Без буферизации такие ходы игнорируются движком, что вызывает раздражение ("игра тупит"). Однако прямая обработка свайпа нестабильных сфер приводит к поломке логики матчей.

## Решение
Внедрить компонент `InputBufferController`, реализующий гибридную модель буферизации ввода:

1. **Разрешённые действия**:
   - Двигать и свайпать полностью стабильные сферы (`CellState == STABLE`).
   - Буферизовать ровно один ход (`max_queue_size = 1`) во время анимации каскадов, если ход затрагивает ячейки, которые сейчас нестабильны (`FALLING`, `RESOLVING`), но освободятся в ближайшее время.

2. **Очередь ходов (InputQueue)**:
   При попытке игрока совершить свайп в нестабильной зоне, ход упаковывается в объект `QueuedMove`:
   ```gdscript
   class QueuedMove:
       var from_cell: Vector2i
       var to_cell: Vector2i
       var created_at: float
       var expires_at: float
       var expected_from_gem_id: int
       var expected_to_gem_id: int
   ```
   В момент свайпа целевые ячейки переводятся в состояние `RESERVED`, чтобы никакие другие ходы не могли их переопределить.

3. **Время жизни буфера (Expiration)**:
   Ход хранится в буфере в течение фиксированного времени `expires_at` (0.3–0.8 сек, настраивается в конфиге). По истечении этого времени, если ячейки не стабилизировались, ход автоматически удаляется из очереди без штрафа для игрока.

4. **Валидация при активации**:
   Как только ячейки `from_cell` и `to_cell` переходят в `STABLE`, система сверяет текущие ID сфер с записанными `expected_from_gem_id` и `expected_to_gem_id`.
   - Если ID совпадают: ход выполняется.
   - Если сферы изменили свой тип после падения: ход удаляется, запускается "мягкий" сброс с визуальным pop-эффектом и легкой вибрацией (без штрафа очков).

## Последствия
- Ощущение "ультра-отзывчивости" игры — ходы игрока не теряются.
- Предотвращение некорректных матчей из-за устаревшего состояния поля.
- Необходимость точного контроля времени жизни буфера (слишком долгое удержание вызывает зависание хода, слишком короткое делает буфер бесполезным).
