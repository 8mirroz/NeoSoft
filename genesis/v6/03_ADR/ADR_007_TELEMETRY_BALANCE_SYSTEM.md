# ADR 007: Телеметрия Баланса и Data-Driven Настройка (Balance Telemetry Layer)

## Статус
Принято (Active)

## Контекст
Внедрение Combo Fever Engine кардинально меняет скорость прохождения уровней и кривую сложности. Без детального логирования показателей игрового процесса во время MVP-тестирования невозможно объективно оценить баланс, win-rate уровней, частоту генерации спец-сфер и эффективность буферизованного ввода.

## Решение
1. **Слой телеметрии BalanceTelemetryLayer**:
   Внедряется легковесный класс `BalanceTelemetryLayer`, который собирает статистику в реальном времени во время игровой сессии:
   - `average_combo_length`: средняя длина цепочки комбо.
   - `max_combo_length`: максимальная цепочка комбо.
   - `fever_activation_rate`: количество Fever-активаций на уровень.
   - `special_sphere_creation_rate`: количество созданных спец-сфер по типам.
   - `special_combo_usage_rate`: частота активации спец-комбинаций.
   - `invalid_fast_move_rate`: доля невалидных быстрых ходов игрока.
   - `queued_move_success_rate`: процент успешного выполнения буферизованных ходов.
   - `level_win_rate`: процент побед по уровню.
   - `average_moves_left_on_win`: среднее число оставшихся ходов при победе.
   - `player_idle_time_during_cascade`: время пассивного ожидания игрока.

2. **Локальное логирование в JSON**:
   В конце каждого уровня собранный словарь экспортируется в локальный JSON-лог (с поддержкой экспорта через Settings overlay, унаследованной из v2/v3).

3. **Data-Driven Конфигурация Баланса**:
   Все числовые параметры баланса выносятся в единый файл конфигурации `res://data/combo_fever_config.json`:
   - Время Combo Window для каждого уровня сложности.
   - Коэффициенты продления окна при разных типах матчей.
   - Порог очков ComboPower для Fever Mode.
   - Штрафы по времени за невалидные свайпы.

## Последствия
- Возможность точной калибровки сложности и удержания игроков на основе реальных данных.
- Быстрая балансировка параметров без изменения кода скриптов (через JSON).
- Минимальное влияние на производительность (запись событий в памяти происходит мгновенно, запись на диск — только в конце раунда).
