# ADR 001: Выбор технологического стека

*   **Статус**: Утверждено  
*   **Дата**: 2026-05-26  
*   **Контекст**: Выбор основного игрового движка, языка программирования и платформы экспорта для релиза MVP v1.

---

## Варианты выбора

1.  **Unity (C#)**:
    *   *Плюсы*: Огромный рынок ассетов, отличная поддержка 2D/UI.
    *   *Минусы*: Тяжелые сборки для Web, лицензионные риски, долгая настройка CI/CD.
2.  **Godot Engine 4.5/4.6 (GDScript)**:
    *   *Плюсы*: Чрезвычайно легковесный (Web-билд весит <5MB), открытый исходный код (MIT), мгновенный запуск в headless-режиме для тестов, встроенный мощный язык GDScript.
    *   *Минусы*: Менее развитая экосистема готовых плагинов монетизации по сравнению с Unity.

---

## Решение

**Использовать Godot Engine 4.5/4.6 с языком программирования GDScript.**

1.  **Главная платформа**: Web-браузеры (default) + Android (Mobile). Это гарантирует мгновенную доступность игры для тестеров без необходимости установки громоздких пакетов.
2.  **Язык разработки**: Чистый **GDScript** для всех игровых модулей. Использование C# исключено из-за ограничений экспорта Godot под Web (C# Web-сборки требуют экспериментальных сред).
3.  **Сборка**: Canonical-пайплайн в **GitHub Actions** (godot-export) для автоматической сборки Web- и Android-пакетов при каждом пуше.

---

## Последствия

*   Все новые геймплейные скрипты должны строго следовать синтаксису GDScript 2.0 (Godot 4.x).
*   Для проведения автотестов на серверах сборки можно использовать headless-запуск Godot (`Godot --headless`), что уже успешно реализовано в E2E-тесте `validate_soft_launch_smoke.gd`.
