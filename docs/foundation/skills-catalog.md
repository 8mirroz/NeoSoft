## SKILL-002: `match3-balance-check`

**Триггер:** новый уровень создан, кривая сложности скорректирована

**Workflow:**
1. Валидировать JSON-схему конфига уровня
2. Проверить ratio ходы/цели
3. Убедиться: gem types ≥ 5, ≤ 8
4. Проверить плотность блокеров
5. Запустить симуляцию скоринга (будущее: MCTS-бот)
6. Сравнить с шаблоном кривой сложности

**Метрики:** `loss_rate`, `fuuu_factor`, `avg_completion_pct`

**Кривая сложности:**
```
L1-25:  Почти беспроигрышные (обучение)
L26-62: Плавный рост
L63-76: Пик сложности (paywall zone)
L77-85: Спад (отдых)
L86+:   Новый рост
```

**Правила:**
- Пейволлы между L40 и L70
- Разница 20%-50% loss rate ощущается слабо
- Каждый % критичен только при >80% сложности
- Первые 3 уровня — самые важные для вовлечения

## SKILL-004: `vfx-review`

**Триггер:** новый VFX создан, изменение каскадного поведения

**Checklist:**
- [ ] Радиус эффекта ≤ 1 клетки
- [ ] Каскадный offset ~0.05s между элементами
- [ ] Нет одновременного VFX sandwich
- [ ] Цветовая логика корректна (нейтральный для generic, цветной для type-specific)
- [ ] Видимость на светлом фоне (подложка добавлена при необходимости)
- [ ] Физика консистентна (вес, bounce, инерция)
- [ ] Разлёт осколков ≤ 1 клетки

**7 лайфхаков (Render.ru/Playrix):**
1. Точное местоположение + плавное затухание
2. Читаемость цвета (нейтральный по умолчанию)
3. Временные офсеты (1-2 кадра)
4. Приоритеты (фишка летит в цель → нет dissolve)
5. Контекст (эффект виден под глобальным взрывом)
6. Физика (желе ≠ камень, два направления гравитации)
7. Ресурсы (базовые = идеал, редкие = меньше)

## SKILL-005: `level-design-from-notebook`

**Триггер:** планирование батча уровней (10+)

**Workflow:**
1. Запросить NotebookLM рекомендации по кривой сложности
2. Применить волновой паттерн (easy → hard → rest → hard)
3. Привязать механики к диапазонам уровней:
   - L1-10: swap, match 3, match 4, цели, бустеры
   - L11-30: лёд, стекло, пузырьки, замороженные ячейки
   - L31-60: несколько целей, ограниченные ходы, спецгемы
   - L61+: нестандартные формы, туман, цепи, drop-objects
4. Установить paywall zone (L40-L70)
5. Сгенерировать JSON-конфиги
6. Запустить симуляцию баланса

**Правила:**
- Первые 3 уровня показывают «фишку» игры
- Сложные механики (drop) не в первых уровнях
- Не смешивать 3-4 режима в одном уровне
- Стандартное очко match 3 = 60 (20 × 3), +10 за доп. фишку

## SKILL-006: `ethical-monetization-audit`

**Триггер:** любая фича монетизации, дизайн магазина, размещение рекламы

**Checklist:**
- [ ] Нет скрытых вероятностей
- [ ] Нет фейковых якорных цен
- [ ] Нет свинки-копилки
- [ ] Нет эскалации стоимости «+5 ходов»
- [ ] Нет давления на покупку через фрустрацию
- [ ] Rewarded ads = добровольные
- [ ] Cosmetics не дают gameplay power
- [ ] Нет манипулятивного UI (≤2 клика до отмены)
- [ ] Цены не заканчиваются на «9» с фейковой скидкой

**Этичные альтернативы:**
- Визуализация прогресса (шкалы, трофеи)
- Коллекционирование как long-term hook
- Daily reward без жёстких последовательностей
- Сезонные темы без hard deadline

## SKILL-007: `continuous-gravity-implementation`

**Триггер:** реализация ядра gameplay (Phase B)

**Принципы:**
1. Каждый кадр одновременно обновляются: swap, match, fall, boosters
2. Игрок может собирать комбинации в незатронутых колонках
3. Удары/свапы привязаны к координатам сетки (homes), не к физическим плиткам

**Edge cases:**
- Падающие плитки образуют match «в воздухе» → решение через homes
- Летящая ракета задевает падающие элементы → проверка по grid position
- Новый swap во время незавершённого каскада → queue management

**Архитектура:**
```
State Machine (FSM) → LogicState.onTick() → Systems → Atomic Actions
                                              ├─ MatchSystem
                                              ├─ SheddingSystem (гравитация)
                                              └─ GenerationSystem (спавн)
```
