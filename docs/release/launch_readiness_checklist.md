# Чек-лист готовности к релизу (Launch Readiness Checklist) — Genesis v5.1

Документ описывает строгий набор Go/No-Go критериев, которые должны быть выполнены перед сборкой релизных пакетов (APK/IPA) игры **Neo Soft Frost**.

---

## 1. Автоматизированные критерии (CI Gates)

* [ ] **GUT Тесты**: Все юнит- и интеграционные тесты детерминированного ядра в папке `tests/core_match3/` проходят без ошибок (`run_headless_tests.sh` возвращает exit code 0).
* [ ] **Регрессионные реплеи**: Все реплеи из папки `artifacts/replays/regression/` воспроизводятся без рассинхронизаций (desync).
* [ ] **E2E MCTS баланс**: Headless-симулятор MCTS подтверждает сходимость уровней сложности (средняя глубина каскада $\le 3.2$, win-rate соответствует профилю).
* [ ] **Встроенный бенчмарк**: FPS-тест бюджетов частиц не опускается ниже 55 кадров/сек при пиковых взрывах на SAFE-профиле.

---

## 2. Архитектурные и механические правила (Architecture & Invariants)

* [ ] **Отсутствие бесконечных каскадов**: CascadeGovernor гарантированно останавливает бесконечные цепные реакции на шаге 10.
* [ ] **Изоляция слоев**: В коде папки `scripts/core_match3/` отсутствуют перекрестные импорты визуальных нод или HUD.
* [ ] **Анти-фриз FSM**: ResolvePipelineTimer автоматически сбрасывает поле в стабильное состояние при любых непредвиденных зависаниях анимаций дольше 2 секунд.
* [ ] **Безопасность ассетов**: Удаление любого медиа-файла из манифеста ассетов не приводит к вылету игры, а корректно обрабатывается через fallback-заглушку.
