# Полный план разработки игры Match-3 в стиле Neo Soft Frost

## 0. Цель проекта

Разработать **премиальную визуальную игру “3 в линию”** с нестандартной эстетикой: не candy-style, не яркие фрукты/кристаллы, а **прозрачные световые сферы-гемы** в стиле:

**Neo Soft Frost / glassmorphism / liquid crystal / soft realistic minimalism.**

Игра должна ощущаться как спокойная, красивая, тактильная, визуально дорогая головоломка с большим акцентом на:

* прозрачные 3D/2.5D сферы;
* мягкую физику;
* световые эффекты;
* динамическое освещение;
* жидкостные анимации;
* приятный интерфейс;
* плавные match-3 взаимодействия;
* мобильный формат.

---

# 1. Базовая концепция игры

## Жанр

**Match-3 puzzle game** — игрок меняет местами соседние гемы, собирает линии из 3+ одинаковых элементов, выполняет цели уровня и проходит карту/серии уровней.

## Ключевое отличие

Вместо яркого различия по цветам гемы различаются по:

* внутреннему рисунку;
* типу движения;
* форме светового слоя;
* пузырькам;
* волнам;
* вихрям;
* кольцам;
* ядру;
* направлению блика;
* мягкому оттенку.

Это делает игру более эстетичной и необычной.

## Рабочее название

**Neo Soft Frost**

Варианты названий:

* Soft Orb
* Frost Lines
* Glass Match
* Pearl Flow
* Liquid Light
* Aura Gems
* Neo Frost Puzzle
* Sphere Garden
* Flow Match
* Crystal Calm

# 2. Основной игровой цикл

## Базовый цикл уровня

1. Игрок видит поле 8×8 или больше.
2. На поле расположены сферы разных типов.
3. Игрок выбирает сферу и меняет ее с соседней.
4. Если образуется линия из 3+ одинаковых сфер — они исчезают.
5. На их место падают новые сферы.
6. Срабатывают каскады.
7. Игрок выполняет цель уровня.
8. Получает звезды, очки, награду.
9. Переходит к следующему уровню.

## Основная формула

**Выбор → swap → match → dissolve → fall → spawn → cascade → reward**

# 3. Игровое поле

## Базовое поле

Рекомендуемый старт:

**8×8**

Это стандартный безопасный вариант для MVP.

## Нестандартное расширенное поле

Так как ты хочешь нестандартное поле большего размера, можно использовать:

### Вариант A — 9×9

Лучший баланс. Поле больше классики, но не перегружает экран.

### Вариант B — 10×8

Шире классического поля, хорошо смотрится на планшете и desktop.

### Вариант C — 10×10

Красиво и масштабно, но сложнее для мобильного UX.

### Вариант D — динамическое поле

На разных уровнях меняется форма:

* 8×8 классика;
* 9×9 расширенное;
* 10×8 горизонтальное;
* поле с заблокированными ячейками;
* поле с отверстиями;
* поле с зонами льда;
* поле с энергетическими каналами.

## Рекомендация

Для MVP:

**8×8 + подготовить систему под 9×9 и 10×8.**

Архитектурно нельзя жестко зашивать размер поля. Нужно делать board config:

```json
{
  "width": 8,
  "height": 8,
  "cellSize": 96,
  "layout": "rect",
  "blockedCells": []
}
```

# 5. Специальные гемы

## 5.1. Line Blast Gem

Создается при совпадении 4 в ряд.

Эффект:

* горизонтальная или вертикальная световая волна;
* уничтожает всю строку или колонку;
* визуально выглядит как сфера с яркой внутренней лентой.

Варианты:

* `line_horizontal`
* `line_vertical`

## 5.2. Pulse Bomb Gem

Создается при совпадении T- или L-формы.

Эффект:

* взрыв мягкой световой волной;
* очищает радиус 3×3;
* визуально — сфера с ярким ядром.

## 5.3. Prism Gem

Создается при совпадении 5 в ряд.

Эффект:

* удаляет все сферы выбранного типа;
* визуально — прозрачная радужная призматическая сфера.

## 5.4. Cross Wave Gem

Создается при сложном совпадении 5+ с пересечением.

Эффект:

* очищает строку и колонку одновременно;
* визуально — сфера с крестовым свечением.

## 5.5. Gravity Orb

Особый гем для поздних уровней.

Эффект:

* притягивает сферы вокруг;
* меняет расположение объектов;
* подходит для уникальной механики игры.

# 6. Препятствия и игровые объекты

Чтобы игра не была однообразной, нужны препятствия.

## Базовые препятствия

### Ice Layer

Ледяная пленка поверх сферы.
Нужно собрать match рядом или прямо на ней.

Уровни:

* ice_1 — тонкий слой;
* ice_2 — двойной слой;
* ice_3 — плотный матовый слой.

### Glass Lock

Сфера заперта в стеклянной оболочке.
Нельзя двигать, пока не разбить.

### Frost Tile

Замороженная ячейка поля.
Гем может стоять на ней, но нужно очищать фон.

### Bubble Blocker

Пузырь-блокер занимает ячейку.
Уничтожается матчем рядом.

### Mist Cell

Туманная ячейка скрывает гем.
При совпадении рядом туман рассеивается.

### Crystal Chain

Тонкая световая цепь вокруг сферы.
Гем нельзя двигать, пока цепь не разрушена.

# 7. Цели уровней

## Основные цели

### Collect Gems

Собрать нужное количество определенных сфер.

Пример:

* собрать 20 Blue Flow;
* собрать 15 Violet Core;
* собрать 10 Aqua Bubble.

### Clear Frost

Очистить замороженные клетки.

### Break Glass

Разбить стеклянные блокеры.

### Drop Object

Опустить специальный объект вниз поля.

Например:

* световая капля;
* ключ-сфера;
* фрагмент кристалла.

### Score Target

Набрать нужное количество очков.

### Combo Target

Сделать определенное количество комбо.

### Limited Moves Challenge

Выполнить цель за ограниченное количество ходов.

# 8. Система уровней

## Структура уровня

Каждый уровень должен храниться как конфиг.

Пример:

```json
{
  "levelId": 1,
  "board": {
	"width": 8,
	"height": 8
  },
  "moves": 25,
  "goals": [
	{
	  "type": "collect",
	  "gemType": "blue_flow",
	  "count": 20
	}
  ],
  "gemTypes": [
	"frost_pearl",
	"clear_glass",
	"blue_flow",
	"violet_core",
    "mint_wave"
  ],
  "blockers": [],
  "difficulty": 1
}
```

## Прогрессия сложности

### Уровни 1–10

Обучение:

* swap;
* match 3;
* match 4;
* цели;
* бустеры.

### Уровни 11–30

Ввод препятствий:

* лед;
* стекло;
* пузырьки;
* замороженные ячейки.

### Уровни 31–60

Комбинации:

* несколько целей;
* ограниченные ходы;
* спецгемы;
* каскады.

### Уровни 61+

Сложные паттерны:

* нестандартные формы поля;
* туман;
* цепи;
* drop-objects;
* редкие гемы.

# 14. Анимации гемов

## 14.1. Idle

Постоянная мягкая анимация.

Для каждого типа гема:

* движение внутренней волны;
* мягкая пульсация;
* shimmer блика;
* легкий breathing scale;
* micro glow.

## 14.2. Select

Когда игрок выбирает гем:

* увеличение на 3–5%;
* световое кольцо;
* усиленный контур;
* короткий pulse.

## 14.3. Deselect

Сфера возвращается в нормальное состояние.

## 14.4. Swap Valid

Если ход успешный:

* две сферы меняются местами;
* световой trail;
* elastic finish;
* маленький ripple.

## 14.5. Swap Invalid

Если совпадения нет:

* сферы меняются и возвращаются обратно;
* мягкий shake;
* красный цвет не использовать, лучше легкий серый pulse.

## 14.6. Match 3

* сферы светятся;
* растворяются;
* исчезают через прозрачный dissolve.

## 14.7. Match 4

* одна сфера превращается в line blast;
* по ней проходит световая лента.

## 14.8. Match 5

* появляется Prism Gem;
* радужный импульс;
* мягкая вспышка.

## 14.9. Fall

* сферы падают сверху;
* движение с easing;
* небольшой bounce при посадке.

## 14.10. Spawn

* сфера собирается из света;
* сначала glow;
* потом оболочка;
* потом внутренний рисунок.

# 20. Алгоритм хода

## Последовательность

1. Игрок нажимает на гем.
2. Система сохраняет selected gem.
3. Игрок нажимает на соседний гем.
4. Проверяется соседство.
5. Выполняется swap.
6. Проверяется наличие match.
7. Если match нет — swap обратно.
8. Если match есть:

   * списать ход;
   * найти все совпадения;
   * удалить совпавшие гемы;
   * начислить очки;
   * обновить цели;
   * применить гравитацию;
   * создать новые гемы;
   * проверить каскады.
9. Проверить победу.
10. Проверить поражение.

# 21. Генерация поля

## Важно

Стартовое поле не должно иметь готовых совпадений.

Алгоритм:

1. Заполнить ячейки случайными гемами.
2. Для каждой новой сферы проверять:

   * не создает ли она match 3 по горизонтали;
   * не создает ли она match 3 по вертикали.
3. Если создает — выбрать другой тип.
4. После заполнения проверить, есть ли возможные ходы.
5. Если ходов нет — перегенерировать или перемешать.

Если игрок не делает ход 5–7 секунд:

* мягко подсветить возможную пару;

# 29. Анимационные ассеты

## Для каждого гема

Можно использовать 2 подхода.

### Подход A — статичный PNG + shader animation

Лучший вариант.

Сфера — PNG, а переливы/свет делаются shader-слоями.

Плюсы:

* меньше вес;
* гибкая анимация;
* высокое качество;
* можно делать разные состояния.

### Подход B — видео/WebM/PNG sequence

Для премиальных idle-анимаций.

Минусы:

* большой вес;
* сложнее оптимизация;
* дорого для мобильного.

## Рекомендация

Для MVP:

**PNG + shader + particle FX.**

# 39. Главные риски

## Риск 1. Гемы плохо различаются

Решение:

* усилить разные паттерны;
* добавить уникальные внутренние формы;
* добавить разные анимационные ритмы;
* использовать не только цвет, но и структуру.

## Риск 2. Белые сферы теряются на светлом фоне

Решение:

* добавить мягкий контур;
* усилить ambient occlusion;
* использовать board shadow;
* добавить subtle glow separation.

## Риск 3. Игра выглядит красиво, но нечитабельно

Решение:

* тестировать в размере 64×64 и 96×96;
* каждый гем должен быть узнаваемым маленьким;
* не делать слишком много похожих прозрачных вариантов.

## Риск 4. Слишком тяжелая графика

Решение:

* PNG/WebP оптимизация;
* shader вместо видео;
* atlas textures;
* отключаемые FX;
* quality settings.

## Риск 5. Match-3 без уникальной механики

Решение:

* добавить различие по динамике сфер;
* сделать light-flow комбо;
* добавить glass/frost препятствия;
* сделать zen/premium настроение;
* развить коллекцию сфер.

# 40. Финальная продуктовая формула

Игра должна продаваться не как обычная match-3, а как:

**премиальная медитативная puzzle-игра со стеклянными живыми сферами, мягкой физикой света и визуальным удовольствием от каждого касания.**

Короткая формула:

**Match-3 + glassmorphism + liquid light + calm premium puzzle.**

Главное в разработке:

1. Сначала сделать идеальную механику.
2. Потом добиться читаемости гемов.
3. Потом довести анимации до “дорогого” ощущения.
4. Потом строить уровни, удержание и монетизацию.
5. Не потерять минимализм и легкость.
