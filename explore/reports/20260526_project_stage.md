## 2. Key Insights

1. **Проект находится на стадии завершения MVP (Phase D) и подготовки к Soft Launch (Phase F)**: Все базовые игровые механики, 10 уровней с автобалансировкой, сохранение прогресса и автотесты полностью готовы и верифицированы.
2. **Инновационный векторный рендеринг фишек (GemView)**: Визуальный стиль "Neo Soft Frost" (матовое стекло, жидкостные волны, внутреннее свечение, дрейфующие пузырьки) реализован с помощью чистого векторного рисования (`_draw()`), что обеспечивает кристальную четкость при высокой энергоэффективности на мобильных и Web-платформах.
3. **Высочайшая степень автоматизации QA**: Внедрен автоматический сквозной дымовой тест (`SoftLaunchSmokeRunner`), проверяющий все сценарии от запуска до завершения уровня, а также MCTS-симуляция прохождения 10 уровней.
4. **Блокирующие факторы локального экспорта**: Сборка Web и Android билдов локально заблокирована из-за отсутствия экспортных шаблонов Godot 4.5.1 и путей к Android/Java SDK на данной машине. Canonical-путь сборки через GitHub Actions полностью готов.

## 3. Detailed Findings

### 3.1 Анализ планов и прогресса из документации

**Exploration Method**: 🔍 Outward Search (анализ файлов документации)

**Findings**:
- **[DEVELOPMENT_PIPELINE.md](file:///Users/user/3-line/DEVELOPMENT_PIPELINE.md)**: Задает дорожную карту из 6 фаз (A–F). Проект успешно преодолел Pre-production (A), Prototype (B) и Visual Integration (C). Сейчас он находится на стыке MVP Production (D) и Soft Launch Prep (F).
- **[p1.md](file:///Users/user/3-line/p1.md)**: Описывает визуальный стиль "Neo Soft Frost" (Luxury Glassmorphism, портретное разрешение 720×1280, пастельные мягкие тона, 10 различных типов прозрачных сфер с индивидуальной внутренней динамикой).
- **[p2.md](file:///Users/user/3-line/p2.md)**: Детализирует геймплейные правила (сетка 8×8, специальные гемы, препятствия вроде льда и стеклянных блоков, 10 стартовых уровней, 3 внутриигровых бустера: Hammer, Shuffle, Undo).
- **[libra.md](file:///Users/user/3-line/libra.md)**: Фиксирует лицензирование (MIT/CC0) и внешние интеграции (Kenney Starter Kit Match-3 для механики поля + Maaack Godot Game Template для оболочки).
- **[mvp-checklist.md](file:///Users/user/3-line/docs/foundation/mvp-checklist.md)**: Показывает, что 100% требований геймплея и UI MVP выполнены. Не закрыты только локальные тесты сборки.

### 3.2 Конфигурация Godot и Архитектура

**Exploration Method**: 🔍 Outward Search (анализ проекта и кода)

**Findings**:
- **Система Autoloads (Синглы)**:
  - `EventBus`: Обеспечивает полностью событийно-ориентированную развязку логики и представления (RULE-005).
  - `UserData`: Отвечает за локальное сохранение (`user://save_data.cfg`), прогресс уровней, монеты и инвентарь бустеров.
  - `AnalyticsTracker`: Записывает 9 ключевых продуктовых событий (начало сессии, старт/финиш уровня, использование бустеров/отмены и т.д.) в формате JSONL по пути `user://analytics_events.jsonl`.
  - `SoftLaunchSmokeRunner`: Автозапуск E2E автотеста при запуске с флагом `--smoke-soft-launch`.
  - `SoundManager`: Управляет звуковым окружением.
- **Дисплей и рендеринг**: Проект настроен под портретный режим 720×1280 со сжатием `canvas_items` (expand), рендеринг выставлен на мобильный (`mobile`), что гарантирует гладкую производительность.

### 3.3 Уровень зрелости UI и Геймплея (GemView)

**Exploration Method**: 🔍🧠 Mixed (анализ кода отрисовки + визуальная оценка)

**Findings**:
- Скрипт `scripts/presentation/gem_view.gd` представляет собой шедевр векторного рисования:
  - **Эффект стекла и неона**: Реализован через наложение мягких полупрозрачных теней (`Color(0, 0, 0, 0.12)`) и рассеянных ореолов (`Color(..., 0.22)`).
  - **Физика и динамика**: Внедрены микроанимации «дыхания» (плавное масштабирование с циклом ~3.3 сек через `sin(time_elapsed * 1.8)`), уникальные для каждого из 8 типов сфер:
    - *Pink Pearl*: вращающиеся жидкостные волны, блик.
    - *Blue Flow*: ромбовидный гем с вертикальной текучей лентой.
    - *Ice Spark*: кристаллическая звезда с внутренними гранями.
    - *Frost Pearl*: сфера с медленно плывущими газовыми пузырьками.
    - *Mint Shiver*: треугольник с вращающейся дугой ветра.
    - *Gold Aurora*: квадрат с пульсирующим энергетическим ядром.
    - *Amethyst Haze*: восьмиугольник с концентрическими орбитальными кольцами.
    - *Rose Glow*: цветок с вращающимся спиральным ядром.
- **Взаимодействия**: Реализовано выделение (пульсирующее внешнее кольцо), плавный свайп, отмена хода (Undo) с восстановлением состояния, и авто-перемешивание поля (dead-board recovery) при отсутствии возможных ходов.

### 3.4 Текущие блокировки и качество

**Exploration Method**: 🔍 Outward Search

**Findings**:
- **Локальный экспорт заблокирован**:
  - Отсутствуют экспортные шаблоны Godot версии `4.5.1.stable` (локальная папка `~/Library/Application Support/Godot/export_templates/` пуста или не содержит нужную версию).
  - Не сконфигурированы пути к Android SDK и Java SDK (JDK) в настройках редактора Godot.
- **Решение**: Использовать каноничный билд-пайплайн в GitHub Actions (`.github/workflows/`), который полностью настроен под автоматический экспорт Web и Android.
